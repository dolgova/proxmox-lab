# Environment Reference — Proxmox Private Cloud Lab

Complete configuration for the single-laptop Proxmox lab. Read this before generating
any Terraform or Ansible code.

---

## Network Layout

| Host | IP | Role |
|---|---|---|
| Proxmox VE host | 192.168.1.100 | Hypervisor (VirtualBox VM) |
| Active VM (any node) | 192.168.1.101 | Web node — only one runs at a time |
| Proxmox Web UI | https://192.168.1.100:8006 | Management interface |

All VMs share the same IP (192.168.1.101) because only one runs at a time.
The bridge interface is `vmbr0`.

---

## Hardware Constraints

| Resource | Value | Note |
|---|---|---|
| Total RAM | 7.7 GB | Proxmox uses ~1.6 GB |
| VM RAM (default) | 256 MB | Configurable in `terraform/variables.tf` |
| Max running VMs | 1 | Running 2+ crashes Proxmox under NEM emulation |
| CPU cores per VM | 1 | Default — adjustable |
| KVM acceleration | Disabled | Must run `qm set <vmid> --kvm 0` before starting |

**Why KVM is disabled:** The Proxmox host runs inside VirtualBox. Nested KVM is not
available when Hyper-V is active on Windows. NEM (Native Execution Mode) is used instead,
which is slow but functional.

## Golden Image (VM 100)

| Item | Value |
|---|---|
| Template VM ID | 100 |
| Name | `golden-template` |
| OS | Alpine Linux 3.19 |
| User | `root` |
| Password | `Alpine2026` |
| SSH | Pre-configured with sshd enabled |
| Packages | Base Alpine + OpenSSH (pre-baked, no runtime installs) |

**Do NOT clone from VM 9000** — that is the raw cloud image. VM 100 has SSH keys,
password auth, and sshd pre-configured.

### Golden Image Rebuild

If you need to rebuild VM 100:
1. Clone VM 9000 to a new VM
2. Start the new VM via Proxmox console (not SSH — no network yet)
3. Configure: set root password, install openssh, enable sshd, configure network
4. Shut down and convert to template
5. Update clone source in `terraform/main.tf` to point to the new template ID

## Ansible Inventory Format

Auto-generated by `scripts/scale.sh`:

```ini
[web_nodes]
web-node-1 ansible_host=192.168.1.101 ansible_user=root ansible_ssh_private_key_file=~/.ssh/proxmox-lab ansible_ssh_common_args='-o StrictHostKeyChecking=no'

[web_nodes:vars]
ansible_python_interpreter=/usr/bin/python3
```

Note: `ansible_python_interpreter` is set but Python is not actually present.
Standard modules will fail — always use the `raw` module.
