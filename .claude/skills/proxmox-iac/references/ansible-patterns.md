# Ansible Patterns — Proxmox Lab

Correct patterns for managing Alpine Linux VMs with no Python and no internet.
Read before generating any Ansible code.

---

## Golden Rule: Raw Module Only

Standard Ansible modules require Python on the target. These VMs have no Python
and no internet to install it. **Every task must use the `raw` module.**

```yaml
# CORRECT — works without Python
- name: Check hostname
  raw: hostname

# WRONG — requires Python
- name: Check hostname
  command: hostname

# WRONG — requires Python
- name: Copy file
  copy:
    src: myfile.conf
    dest: /etc/myfile.conf

# WRONG — requires Python
- name: Install package
  apk:
    name: nginx
    state: present
```

## Playbook Structure

```yaml
# ansible/playbook.yml

- name: Configure web nodes
  hosts: web_nodes
  gather_facts: false    # REQUIRED — facts need Python
  roles:
    - motd
    - ssh-keys
    - health-check
```

**Always set `gather_facts: false`** — Ansible's fact gathering requires Python.

---

## Role Structure

```
ansible/roles/{rolename}/
├── tasks/
│   └── main.yml     # All tasks using raw module
└── README.md        # What the role does
```

Do NOT create: `templates/`, `files/`, `vars/`, `handlers/` — these directories
require Python-based modules to process.

## Task Patterns

### Write a file (replacement for template/copy)

```yaml
- name: Deploy config file
  raw: |
    cat > /etc/myapp.conf << 'CONF'
    setting1=value1
    setting2=value2
    hostname={{ inventory_hostname }}
    CONF
```

Use heredoc (`<< 'CONF'`) for multi-line content. Single-quoted delimiter
(`'CONF'`) prevents shell expansion — use unquoted (`<< CONF`) when you want
variable substitution.

### Manage a service (replacement for service module)

```yaml
- name: Start and enable sshd
  raw: |
    rc-service sshd start 2>/dev/null || true
    rc-update add sshd default 2>/dev/null || true

- name: Restart a service
  raw: rc-service sshd restart

- name: Check service status
  raw: rc-service sshd status
```

**Alpine uses OpenRC**, not systemd. Commands:
- `rc-service <name> start|stop|restart|status`
- `rc-update add <name> default` (enable at boot)
- `rc-update del <name> default` (disable at boot)

### Create directories and set permissions

```yaml
- name: Create app directory
  raw: mkdir -p /opt/myapp && chmod 755 /opt/myapp && chown root:root /opt/myapp
```

### Conditional logic

```yaml
- name: Add line only if not present
  raw: |
    grep -q 'my_setting' /etc/myapp.conf || echo 'my_setting=value' >> /etc/myapp.conf
```

### Check and report

```yaml
- name: Health check
  raw: |
    echo "=== $(hostname) ==="
    echo "Uptime: $(uptime)"
    echo "IP: $(ip addr show eth0 | grep 'inet ' | awk '{print $2}')"
    echo "Memory: $(free -m | awk '/^Mem:/{print $3"/"$2" MB"}')"
    echo "Disk: $(df -h / | awk 'NR==2{print $3"/"$2}')"
```

### Using Ansible variables

Ansible variables (like `inventory_hostname`) work even without Python because
they're expanded by the Ansible controller before sending to the target:

```yaml
- name: Set hostname banner
  raw: "echo 'Node: {{ inventory_hostname }}' > /etc/motd"
```

**But `ansible_facts` don't work** — they require `gather_facts: true` which needs Python.
Use shell commands instead:

```yaml
# WRONG — needs gather_facts
- name: Show IP
  raw: "echo {{ ansible_default_ipv4.address }}"

# CORRECT — use shell
- name: Show IP
  raw: "ip addr show eth0 | grep 'inet ' | awk '{print $2}'"
```

## Running Ansible

```bash
# Run full playbook
cd ansible
ansible-playbook -i inventory.ini playbook.yml

# Run specific role by tag
ansible-playbook -i inventory.ini playbook.yml --tags motd

# Ad-hoc command (no role needed)
ansible web_nodes -i inventory.ini -m raw -a "hostname"

# Test connectivity
ansible web_nodes -i inventory.ini -m raw -a "echo ok"
```

**Do NOT use `ansible ... -m ping`** — the ping module requires Python.
Use `-m raw -a "echo ok"` instead.

## Inventory Format

Auto-generated by `scripts/scale.sh`:

```ini
[web_nodes]
web-node-1 ansible_host=192.168.1.101 ansible_user=root ansible_ssh_private_key_file=~/.ssh/proxmox-lab ansible_ssh_common_args='-o StrictHostKeyChecking=no'

[web_nodes:vars]
ansible_python_interpreter=/usr/bin/python3
```

Only one node is in the inventory at a time (the active one). The IP is always
192.168.1.101 regardless of which VM is running.

## Common Mistakes

| Mistake | Why it fails | Fix |
|---|---|---|
| Using `template` module | Needs Python | Use `raw` with heredoc |
| Using `copy` module | Needs Python | Use `raw` with `cat >` |
| Using `service` module | Needs Python | Use `raw` with `rc-service` |
| Using `file` module | Needs Python | Use `raw` with `mkdir`/`chmod` |
| Using `apt`/`apk` module | Needs Python + internet | Pre-bake in golden image |
| `gather_facts: true` | Needs Python | Always `gather_facts: false` |
| `systemctl` commands | Alpine uses OpenRC | Use `rc-service` / `rc-update` |
| `-m ping` for connectivity | Needs Python | Use `-m raw -a "echo ok"` |
